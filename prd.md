# PRD：《榴莲人格导购官》

## 1. 产品一句话

**用 SBTI 式互动测试帮年轻人选择适合自己的第一颗榴莲，并为商家生成用户画像、导购推荐、直播话术和黑暗品数字名片。**

---

## 2. 项目背景与调研结论

### 2.1 榴莲消费的核心矛盾

榴莲对年轻用户有吸引力，也有明显消费门槛。用户常见顾虑包括：

| 用户顾虑             | 对应业务问题    |
| ---------------- | --------- |
| 不知道哪个品种适合自己      | 导购成本高     |
| 怕买到夹生、坏果、少肉果     | 信任成本高     |
| 觉得榴莲贵，不敢随便试      | 首单转化难     |
| 不会判断成熟度、出肉率      | 教育成本高     |
| 黑暗品 / 非标果看起来不值得买 | 非标果转化难    |
| 直播间气氛很热，但自己不会判断  | 冲动消费后容易售后 |

抖音电商在 2026 年“严选榴莲季”中强调了榴莲品质标准、产地溯源直播、冷链物流、达人内容和消费体验，这说明榴莲电商正在从“低价刺激”走向“品质信任 + 内容导购 + 体验保障”。其中严选榴莲需满足保出四房、出肉率 30% 以上、签收 3 天内包熟等标准。([新华报业网][2])

### 2.2 年轻人为什么愿意玩测试

SBTI 火起来的原因不在于心理学严肃性，关键在于它具备 **低门槛答题、荒诞标签、自嘲表达、结果截图传播** 这些机制。它把 MBTI 式人格分类改造成更适合社交平台传播的玩梗内容。([环球时报][1])

这给本项目一个启发：

**榴莲导购不能一上来就教育用户，要先让用户愿意参与。**

所以本产品用“人格测试”做入口，用“榴莲推荐”做承接，用“商家画像和营销素材”做项目方价值。

### 2.3 AI 内容不能停留在批量生成

小红书 2026 年公布 AI 治理主张，鼓励 AI 作为“创意放大器”，强调真实性、原创价值和真实信息增量，同时反对 AI 造假与低质内容生产。([经济参考网][3])

所以本项目里的 AI 生成不能只做“榴莲发疯文案”。更合理的设计是：

**基于真实用户偏好、榴莲品种信息、商家库存信息，生成导购解释、直播话术和分享卡片。**

这样 AI 负责放大真实信息，避免变成空洞整活。

### 2.4 AI Agent 趋势可以加强技术感

2026 年 AI 应用进入大众化阶段，相关报道显示中国 AI 原生 App 月活用户规模已达到数亿级，AI Agent 也成为互联网公司布局重点。([新浪财经][4])

因此，本项目可以在 Demo 里设计成轻量 Agent 架构：

| Agent     | 作用                |
| --------- | ----------------- |
| 测试 Agent  | 分析用户答题结果          |
| 推荐 Agent  | 匹配榴莲品种和购买场景       |
| 营销 Agent  | 生成直播话术、短视频标题、种草文案 |
| 黑暗品 Agent | 给非标榴莲生成数字名片       |
| 数据 Agent  | 汇总用户画像和商家建议       |

# 5. 核心问题定义

## 5.1 用户侧问题

用户面对榴莲时，不只是在选择水果，也在处理三种不确定性：

| 不确定性  | 表现                 |
| ----- | ------------------ |
| 口味不确定 | 不知道自己适合甜糯、苦甜、奶香、酒香 |
| 品质不确定 | 不知道会不会夹生、坏果、少肉     |
| 社交不确定 | 怕买了没人吃，怕宿舍嫌臭，怕花钱翻车 |

## 5.2 商家侧问题

商家卖榴莲时，核心问题不是单纯曝光，而是：

| 问题      | 表现                  |
| ------- | ------------------- |
| 用户教育成本高 | 每个用户都问“甜不甜、熟不熟、怎么挑” |
| 内容生产成本高 | 直播话术、短视频文案、商品页反复写   |
| 用户偏好难沉淀 | 互动完没有结构化数据          |
| 非标果难转化  | 黑暗品、丑果、裂口果需要重新包装    |
| 售后风险高   | 用户预期和实际开果结果不一致      |

## 8.2 模块二：榴莲人格测试

### 目标

通过轻量问题收集真实导购变量。

### 题目数量

MVP 做 10 题。

### 题目设计原则

每道题都不能只为了搞笑，背后要对应一个推荐标签。

| 题目维度  | 采集目的               |
| ----- | ------------------ |
| 甜度偏好  | 推荐金枕 / 猫山王 / D24   |
| 苦味接受度 | 判断是否适合进阶品种         |
| 臭味接受度 | 判断新手 / 老饕          |
| 预算    | 推荐鲜果 / 冷冻果肉 / 拼团   |
| 场景    | 宿舍 / 礼品 / 独享 / 直播间 |
| 风险偏好  | 是否能接受黑暗品           |
| 决策方式  | 内容种草 / 熟人推荐 / 主播推荐 |
| 售后敏感度 | 是否需要严选信任卡          |
| 分享意愿  | 是否推结果卡             |
| 口味探索欲 | 是否推荐小众品种           |

### 示例题目

| 题目             | A     | B       | C         | D       |
| -------------- | ----- | ------- | --------- | ------- |
| 你买榴莲最怕什么？      | 夹生    | 太贵      | 味太冲       | 不知道怎么挑  |
| 你能接受的榴莲味道是？    | 奶香微臭  | 正常榴莲味   | 越臭越上头     | 我想先轻量尝试 |
| 你更像哪种消费人格？     | 看测评再买 | 朋友推荐就买  | 直播间气氛到位就买 | 只买性价比   |
| 如果榴莲长得丑但便宜，你会？ | 完全不考虑 | 便宜很多可以  | 有故事就想试    | 越怪越想买   |
| 你吃榴莲的场景是？      | 宿舍拼团  | 一个人奖励自己 | 送人        | 直播间冲动下单 |

| 夹生打工人   | 预算敏感、怕踩坑   | 拼团 / 冷冻果肉 | 性价比转化  |
| 爆浆恋爱脑   | 情绪消费、爱送礼   | 礼盒 / 甜品   | 礼品场景   |
| 裂口发疯人   | 爱抽象、能接受非标  | 黑暗品 / 丑果  | 非标果转化  |
| 冷冻理智派   | 怕麻烦、重便利    | 冷冻榴莲肉     | 即食转化   |

## 8.4 模块四：榴莲推荐系统

### 目标

让人格测试结果真正落到导购。

### 推荐输入

| 输入    | 来源  |
| ----- | --- |
| 甜度偏好  | 测试题 |
| 苦味接受度 | 测试题 |
| 臭味接受度 | 测试题 |
| 预算    | 测试题 |
| 食用场景  | 测试题 |
| 风险偏好  | 测试题 |
| 新手程度  | 测试题 |
| 分享意愿  | 测试题 |

### 推荐输出

| 输出     | 示例                   |
| ------ | -------------------- |
| 推荐品种   | 金枕、猫山王、黑刺、D24        |
| 推荐购买形式 | 整果、冷冻果肉、礼盒、拼团        |
| 推荐场景   | 宿舍、礼品、独享、直播间         |
| 推荐理由   | “你偏好甜糯、低风险，所以推荐金枕”   |
| 避坑建议   | “优先选择有包熟保障和出肉率说明的商品” |
| 话术标签   | 新手友好、高甜低风险、适合拼团      |

### MVP 推荐规则

可以先用规则模型，不必一上来做复杂机器学习。

示例：

```text
如果用户甜度偏好高 + 臭味接受度低 + 新手程度高
→ 推荐金枕
→ 文案标签：新手友好 / 甜糯 / 低门槛

如果用户苦味接受度高 + 预算高 + 探索欲强
→ 推荐猫山王或黑刺
→ 文案标签：老饕进阶 / 风味浓郁 / 高端品种

如果用户预算低 + 风险接受度高 + 黑暗品接受度高
→ 推荐黑暗品 / 非标果
→ 文案标签：性价比 / 裂开也很甜 / 丑但能打
```

# 9. 用户故事

## 用户故事 1：第一次买榴莲

作为一个没买过榴莲的大学生，
我想通过一个轻松测试知道自己适合哪种榴莲，
以便降低第一次购买的风险。

### 验收标准

* 用户能在 2 分钟内完成测试
* 结果页给出明确品种推荐
* 结果解释不使用过多专业术语
* 给出至少 1 条避坑建议

## 用户故事 2：宿舍拼团

作为一个想和室友一起买榴莲的人，
我想知道大家的口味是否适合一起拼，
以便选一个大多数人能接受的榴莲。

### 验收标准

* 支持生成分享链接
* 支持多人结果合并
* 输出拼团推荐品种
* 输出人均预算建议

MVP 阶段可以先做“模拟多人结果”，不必开发复杂实时协同。

## 用户故事 3：商家直播

作为一个榴莲主播，
我想根据用户人格生成直播话术，
以便更快回答用户疑问，提高互动。

### 验收标准

* 可选择直播风格
* 可生成开场、答疑、促单、黑暗品话术
* 话术必须包含真实榴莲信息
* 不能只生成空泛段子

## 用户故事 4：黑暗品销售

作为一个有非标榴莲库存的商家，
我想给每颗丑果生成一张数字名片，
以便让用户理解它的特点并愿意尝试。

### 验收标准

* 可输入瑕疵类型
* 可生成拟人化名称
* 可生成适合人群
* 可生成风险说明
* 可生成销售文案

# 19. 项目亮点

## 19.1 亮点一：热点入口

借 SBTI 的传播机制，让榴莲导购从“知识教育”变成“主动参与”。SBTI 的传播特征已经被多家媒体观察到，尤其是截图分享、自嘲标签和年轻人社交表达。([环球时报][1])

## 19.2 亮点二：导购闭环

测试结果不止给标签，还给真实榴莲推荐、购买场景和避坑建议。

## 19.3 亮点三：商家价值

后台能把用户互动转成偏好数据，帮助商家做选品、直播、私域和内容策划。

## 19.4 亮点四：黑暗品转化

通过人格化和数字名片，把非标榴莲从“卖相差”转成“有故事、有适合人群、有购买理由”。

## 19.5 亮点五：可信 AI 内容

生成内容基于用户偏好和榴莲信息，符合“AI 作为创意放大器”的方向，避免空泛批量内容。([经济参考网][3])

# 20. 风险与应对

| 风险        | 表现        | 应对                     |
| --------- | --------- | ---------------------- |
| 太像普通人格测试  | 评委觉得不贴产业  | 加推荐品种、商家后台、黑暗品名片       |
| AI 输出太虚   | 只有段子，没有价值 | 强制输出导购理由和风险提示          |
| 榴莲品种数据不足  | 推荐不够专业    | MVP 使用结构化品种知识库         |
| 分享卡有趣但不转化 | 用户玩完就走    | 结果页加入推荐和拼团入口           |
| 黑暗品包装被质疑  | 可能像掩盖缺陷   | 明确展示缺点、风险和适合人群         |
| 技术范围过大    | 做不完       | 先做规则推荐 + LLM 解释 + 模拟后台 |

# 22. 最终建议版本

如果比赛时间有限，建议你们把产品收敛成这个版本：

## MVP 名称

# 榴莲人格导购官

## MVP 核心

```text
SBTI 测试
+
AI 榴莲推荐
+
分享卡
+
商家画像后台
+
黑暗品数字名片
```

## 不建议第一版做

* 真实图像识别成熟度
* 复杂电商交易
* 多人实时拼团
* 完整库存系统
* 复杂 AI 头像生成

这些可以作为未来规划。

# 23. 答辩结尾可以这样讲

“榴莲消费的难点在于，用户既缺少知识，也缺少信任，还缺少一个低门槛的参与入口。我们用 SBTI 式测试把年轻人吸引进来，用 AI 把测试结果转化为榴莲品种推荐、直播话术和黑暗品数字名片，再用后台沉淀用户画像，帮助商家做选品和营销。这个项目既能被用户玩起来，也能被商家用起来。”

这版 PRD 的核心不是单独做一个测试，而是把测试变成 **榴莲导购和营销的前端入口**。这样“热点、项目方、评分项”就能对齐。
